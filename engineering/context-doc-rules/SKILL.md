# 文档命名与头部规则

## 使用目标

在当前上下文仓库写文档时，先判断文档类型，再只读取对应参考文件，避免一次加载全部规则。

## 执行流程

1. 判断正在写哪类文档。
2. 先读取 `references/通用规则.md`。
3. 再按下表读取一个具体规则文件。
4. 写入前按 `references/最低检查清单.md` 做最终检查。

## 路由表

| 文档类型 | 读取文件 |
| --- | --- |
| 产品需求文档、PRD | `references/产品需求文档规则.md` |
| 工作记录、待办清单、会议记录、整理记录 | `references/工作记录规则.md` |
| 草稿、探索、参考资料、摘录、翻译 | `references/草稿与参考资料规则.md` |
| 已确认上下文文档 | `references/已确认上下文文档规则.md` |
| 运行文件、配置文件、脚本、工具约定入口 | 不使用本技能的文档命名规则，保留项目原有命名约定 |

## 边界

- 只处理文档的命名、标题、头部和更新记录。
- 不决定文档应该写入 `context/`、`workspace/` 还是 `drafts/`；目录分层仍按项目 `AGENTS.md` 判断。
- 不重命名运行文件、配置文件、脚本、`README.md`、`AGENTS.md`、`CLAUDE.md`。
