# 通用规则

## 文件名

文件名必须说明主题，不使用泛名。

不推荐：

```text
整理.md
新文档.md
需求.md
最终版.md
```

推荐：

```text
岗位审阅排序与后续处理状态PRD草案.md
文档命名与头部规则草案.md
技能拆分待办清单-2026-07-03.md
```

时间如果需要出现在文件名里，放在主题后面：

```text
{主题}-{YYYY-MM-DD}.md
```

不使用：

```text
YYYY-MM-DD-{主题}.md
```

## 一级标题

文档一级标题写文档对象，不写泛称。

推荐：

```markdown
# 文档命名与头部规则草案
# 技能拆分待办清单
# 岗位审阅排序与后续处理状态 PRD 草案
```

不推荐：

```markdown
# 文档
# 记录
# TODO
```

标题可以比文件名更适合阅读，但必须和文件主题一致。

## 头部形式

统一使用普通文档信息块，不使用三横线元信息块。

所有时间默认使用东八区，并精确到秒：

```text
YYYY-MM-DD HH:mm:ss UTC+08:00
```
