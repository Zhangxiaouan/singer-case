# 工作记录规则

## 文件名

工作记录、待办清单、会议记录通常和某次讨论或某天进展有关，文件名使用主题后置日期。

推荐：

```text
{主题}待办清单-{YYYY-MM-DD}.md
{主题}会议记录-{YYYY-MM-DD}.md
{主题}整理记录-{YYYY-MM-DD}.md
```

## 头部模板

```markdown
# {主题}

**记录时间**：YYYY-MM-DD HH:mm:ss UTC+08:00  
**记录人**：{智能体名称或用户名称}  
**当前状态**：进行中 / 待确认 / 已完成  
**内容概要**：{一句话说明这份记录保存什么}  
**信息来源**：{用户确认 / 本轮对话整理 / 会议记录 / 旧文档迁移}  

## 更新记录

| 时间 | 更新人 | 更新明细 |
| --- | --- | --- |
| YYYY-MM-DD HH:mm:ss UTC+08:00 | {智能体或用户} | 初始记录。 |
```
