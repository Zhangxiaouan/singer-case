# 产品需求文档规则

## 文件名

产品需求文档必须带版本号。没有版本号时，必须带具体日期。

推荐：

```text
{需求主题}PRD-v{版本号}.md
{需求主题}PRD草案-{YYYY-MM-DD}.md
```

## 头部模板

```markdown
# {需求主题} PRD

**版本**：v{版本号} / 未定，记录日期 YYYY-MM-DD  
**更新时间**：YYYY-MM-DD HH:mm:ss UTC+08:00  
**更新人**：{智能体名称或用户名称}  
**当前状态**：草稿 / 审阅中 / 已确认 / 开发中 / 已上线  
**内容概要**：{1-3 句话说明这份 PRD 要解决什么问题、覆盖什么范围}  
**信息来源**：{用户确认 / 本轮对话整理 / 源项目验证 / 旧文档迁移 / 推断，待确认}  

## 更新记录

| 时间 | 更新人 | 更新明细 |
| --- | --- | --- |
| YYYY-MM-DD HH:mm:ss UTC+08:00 | {智能体或用户} | 初始记录。 |
```
